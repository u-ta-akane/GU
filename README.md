# 使用方法

このアプリはゆるぼのためのBotです。サーバー上でスラッシュコマンドを実行することで利用できます。

### /ゆるぼ
ゆるぼを登録します。/ゆるぼと入力すると、ゲームタイトル、募集人数、募集する日、時、分、ロール、(任意で)ランクという入力欄が出てくると思います。
ランクには自分のランク帯、または募集するランク帯を入力してください。何も書かなければ非ランクのゆるぼとなります。
スラッシュコマンド入力時にそれぞれの欄をタップすれば書き方を見ながら入力できます。また、30分以上先の時間を指定した場合は30分前と時間ぴったりにゆるぼのリマインドが行われます。
なお、出力されたゆるぼには"参加する"ボタンが付いており、これを押下することでゆるぼに参加/離脱できます。

### /delete-ゆるぼ
ゆるぼを削除します。/delete-ゆるぼはオプションで対象のIDを入力します。IDはrefs.Config.YURUBOChannelIDで指定したチャンネルに出力されたゆるぼに記載されています。

以下のaを接頭辞に持つコマンドは管理コマンドであり、後述する内部権限ロールから自動で認可チェックが行われます。(utils.HasAuthority())
### /a-stop-bot
DiscordからBotを終了します。
オプションでログ出力したいテキストを任意で指定できます。
必要権限 : AuthorityBotManagement = 3

### /a-reflash-role-data
refs.RoleMapを更新します。これはBotの稼働中にロールを追加した場合に必要です。
必要権限 : AuthorityReflashData = 4

# 仕様

## 実行方法

&#x20;  初回起動時は・設定が必要な変数を登録するため、全ての設定オプションを使用してください。2回目以降はアプリケーションと同じディレクトリに保存されたsecrets.json、jobData.json、config.jsonから必要なパラメータを取得します。また、実行時はnohupを使用することをおすすめします。

&#x20;

###  オプション 

&#x09;	・設定が必要な変数 の設定オプションの書式は以下の通りです。

--any-option="ENTER INPUT"

### 設定が必要な変数

| 変数名                            | 役割                                               | デフォルト値 | 設定オプション                 |
|--------------------------------|--------------------------------------------------|--------|-------------------------|
| refs.Secrets.BotToken          | Discord Developer Portalで取得した自分のアプリケーションのBotトークン | " "    | --set-bot-token         |
| refs.Config.GuildID            | 対象とするギルド(サーバー)のID                                | " "    | --set-guild-id          |
| refs.Config.ModeratorChannelID | ログを出力するチャンネルのID                                  | " "    | --set-moderator-chan    |
| refs.Config.YURUBOChannelID    | ゆるぼを出力するチャンネルのID                                 | " "    | --set-yurubo-chan       |
| refs.Config.DefaultAuthority   | サーバーに参加したユーザーにデフォルトで与える内部権限ロールのID                | " "    | --set-default-authority |
|

## 内部権限について
管理コマンドの実行にはそれぞれ対応する内部権限を持っている必要があります。これはサーバー上の数字のみで構成されたロール(内部権限ロール)から判定されます。
例えばあるメンバーが"12"というロールを持っていた場合、これは=0b1100より下から3,4ビット目が立っていることになります。すなわち、refs.AuthorityBotManagementとrefs.AuthorityReflashDataをこのメンバーは持っていることになります。

注意 : この仕組み上、メンバーが複数の数字のみで構成されたロールを持つことはできません。そのためサーバー上でメンバーにロール作成の権限を渡してはいけません。

## プロジェクト構成

### utils.Log(err, txt, at)
errにerrorオブジェクト、txtに""とするとエラー出力、errにnil、txtに文字列を入力するとテキスト出力を行います。どちらの場合でもatに使用した関数の名前を書くことでログを見やすくします。出力先はrefs.Config.ModeratorChannelIDで指定したチャンネルと標準出力です。
